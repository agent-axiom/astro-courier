# Astro Courier Contest Submission

**Public GitHub repo:** https://github.com/agent-axiom/astro-courier

**Playable game link:** https://agent-axiom.github.io/astro-courier/

**Short description:**

**Astro Courier** is a cozy-but-chaotic browser space-delivery arcade game where you pilot a tiny courier ship across a colorful solar route, pick up strange cargo, manage fuel, dodge hazards and interceptors, boost through risky lines, dock cleanly, and chase the perfect delivery.

I wanted it to feel instantly playable, but deeper than it first looks: every run is about route mastery, speed, fuel discipline, cargo safety, landing quality, style rewards, medals, best-run chasing, combat pressure, and that "one more delivery" arcade loop.

The game is built as a TypeScript web game with a modular engine/simulation structure, Pixi-powered rendering, contract progression, route feedback, best-run tracking, replay-style run feedback, daily dispatch logic, adaptive game feel, keyboard/gamepad/touch controls, combat interceptors, ship HP, player and enemy fire, and an optional OpenAI-powered Enemy Director running server-side through a Cloudflare Worker. No API keys or secrets are committed; the public build is fully playable from the GitHub Pages link.

Codex helped push the project from a rough idea into a real playable product: architecture, gameplay systems, UI iteration, scoring/readout logic, tests, content structure, deployment workflow, mobile-control iteration, and the optional OpenAI Enemy Director integration. I used Codex not just for isolated snippets, but as an implementation partner for iterating on game feel, clarity, architecture, and production readiness.

Open the link, launch a route, deliver the cargo, then try not to immediately hit "run again."
